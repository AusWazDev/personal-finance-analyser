Toolbar Buttons

These controls appear in the navigation bar at the top of every page.

Button / BadgeWhen to useWhat it does
↑ Upload Files When you have new bank statement files to add Opens a file picker (or drag-and-drop zone). After uploading, the full pipeline runs automatically in the background: files are parsed, deduplicated, auto-categorised using merchant rules, and all reports are regenerated. You can close the modal immediately — processing continues in the background.

Filters & Controls

Period Filter

Select the time range for all dashboard charts and figures. Options include preset periods (This Month, Last Month, This Quarter, Last 3/6/12 Months) and each Australian financial year present in your data. The dashboard loads with the current financial year selected by default.

  • Select Custom Range from the dropdown, then enter from/to dates and click Apply.
  • All cards, tables, and charts update immediately — no page reload needed.
Summary

Summary Cards

Six headline figures across the top of the dashboard:

  • Total Income — all income for the selected period, with a per-month average.
  • Total Expenses — all non-transfer spending, with a per-month average.
  • Net Cashflow — income minus expenses; green when positive, red when negative.
  • Savings Rate — (income − expenses) ÷ income as a percentage. Includes a 12-bar sparkline showing the savings rate for each of the last 12 calendar months (teal ≥20%, amber 0–19%, red negative). Always reflects trailing 12 months regardless of the active period filter.
  • Emergency Fund — how many months of average spending your tagged savings accounts can cover. Colour-coded: teal ≥6 months, amber 3–5 months, red <3 months. The denominator is average monthly non-transfer spend over the last 3 calendar months. If no accounts are marked as savings, the card shows a link to Settings → Accounts where you can tag them.
  • Business Expenses — total spending flagged for reimbursement in the period.

Financial Health Score

A composite 0–100 score that synthesises five aspects of your financial health. Shown below the summary cards on the dashboard. Colour coding: teal ≥75 (Very Good/Excellent), amber 50–74 (Fair/Good), red <50 (Poor).

Component Weight What it measures
Savings Rate30%(income − expenses) ÷ income. ≥20% scores well.
Emergency Fund25%Months of spending covered by savings accounts. ≥6 months scores 100.
Budget Adherence20%% of budget categories on track in the current calendar month.
Debt-to-Income15%Outstanding loan balances ÷ annualised income. Lower is better.
Net Worth Trend10%Whether balance snapshots show net worth rising, flat, or falling month-on-month.
  • Click Show breakdown to see each component's score and contribution, plus personalised improvement tips for any component scoring below 60.
  • The 12-month sparkline shows score history. Emergency fund and debt-to-income use current values for historical months; savings rate and budget adherence are computed per-month from transaction data.
  • If no savings accounts are tagged, the Emergency Fund component defaults to a neutral 50. Tag accounts in Settings → Accounts for an accurate score.

Budget Status

Colour-coded chips showing current-month actual spend vs your configured budget for each category. Updates automatically when the dashboard loads.

  • ✓ Green — under budget.
  • ● Amber — over 80% of budget for the month.
  • ⚠ Red — over budget.
  • Budgets are managed in Settings → Budgets. If no budgets are configured, this panel is hidden.
  • Categories with Rollover enabled carry any unspent balance forward into the next month (capped at the base budget amount). The effective budget shown in the badge already includes the rolled-over amount.

Select a date range — all eight chart tabs update instantly using the embedded transaction data. No server call is made; filtering is client-side and immediate.

Transaction History

All Transactions transactions.html

Complete transaction history across all accounts. Use the search box to filter by description, payee, or note. Use the period, category, account, and business-flag dropdowns to narrow results.

  • Click Change on any row to update its category and sub-category. The sub-category dropdown populates automatically based on the chosen category. Select + Add new sub-category… to add a custom value — it is saved permanently and becomes available across all pages.
  • Click Flags on any row to toggle the BIZ (Business Expense — reimbursable by Bedlin), TAX (Tax Deductible — personal deduction), and GST (GST Claimable — ATO 1/11th rule applies) flags. Flagged badges appear on the transaction row.
  • All edits save automatically — there is no Save button.
  • Click column headers to sort. Results are paginated (100 rows per page) — use the page controls below the table to navigate. Filters apply across all pages.
  • Merchant rules are applied automatically on the next import.
  • Multi-currency: All amounts are stored in AUD. Wise converts at source; Revolut foreign-currency transactions use the AUD balance delta, with the original currency and amount recorded in the Notes column.
  • Transaction splitting: A transaction can be split into multiple child rows (each with its own description, amount, and category). The parent row shows a green SPLIT badge — click the ⋮ button to expand and view the children. Splitting is currently initiated via the API (POST /api/txn/<id>/split); to remove a split, click ✖ Remove split in the expanded view.

Tags /tags

Assign free-form tags to individual transactions for custom grouping (e.g. holiday, tax2025, shared). Tags are separate from categories and can span multiple categories.

  • Click the 🏷 button on any transaction row to open the tag modal — type a tag name and press Enter (or click +) to add it. Click an existing tag chip to remove it.
  • Use the Tag dropdown filter in the transactions page to show only transactions with a specific tag.
  • Open Data → Tags to see a summary page listing all tags with per-tag spend totals and transaction counts.
  • Tags are stored as a comma-separated list in the database; each transaction can have multiple tags.

Review review.html

Transactions auto-categorised as Miscellaneous, grouped by merchant name. Resolving a merchant here updates ALL transactions from that merchant at once.

  • Select a category and optional sub-category from the inline dropdowns on each merchant row — both save instantly. The sub-category dropdown populates based on the selected category.
  • Click the expand arrow (▶) to see individual transactions, add per-transaction notes, or check PayPal match hints.
  • Click Seed Rules from History to scan previously categorised transactions and auto-create merchant rules, so future imports categorise them automatically.
  • Results are paginated (50 groups per page). Filters apply across all pages.
  • Changes are saved instantly. Reports update automatically the next time a file is uploaded.
Transfers & Internal Flows

Transfers transfers.html

Suggested pairs of transactions with equal and opposite amounts within 60 days — likely money moving between your own accounts, loan repayments, or reimbursements.

  • Confirm pairs that are genuine internal transfers to exclude them from spend analysis.
  • Dismiss pairs that are coincidental (two unrelated transactions of the same value).
Reporting

FY Summary fy_summary.html

Income, expenditure, tax estimates, and export options broken down by Australian financial year (1 July – 30 June). Each FY card contains spending tables, tax sections, and download buttons.

  • Each tab covers one financial year — click to switch between years.
  • A KPI banner at the top shows Taxable Income, Total Expenditure, Net Savings, and Savings Rate.
  • Taxable Income = Income + Interest Income only. Board & Lodging (ATO exempt) and Business Reimbursements are excluded and labelled with a badge.
  • An Expenditure by Category table lists all spending categories sorted largest first.
  • The Business Expenses table lists transactions flagged BIZ for employer reimbursement.
  • The Tax-Deductible Expenses table lists transactions flagged TAX — home office, professional memberships, tools.
  • The GST Claimable section lists GST transactions. Claimable GST = 1/11th of total, per ATO standard method.
  • The Dividends & Franking Credits section shows cash dividends, imputation credits, and grossed-up total from the Franking Credits page.
  • The Estimated Income Tax section shows gross tax, LITO offset, Medicare levy, and total payable using ATO brackets for the FY (Stage 3 cuts applied from FY2025). A secondary row shows the adjusted estimate after applying tax-deductible expenses.
  • A HECS/HELP alert appears in amber if your taxable income exceeds the ATO repayment threshold for the year, showing the applicable rate and estimated repayment amount.
  • Download Tax Package — ZIP file containing CSV summaries of GST-claimable, business expenses, tax deductions, capital gains, and franking credits for the year.
  • Xero (OFX) — exports business-flagged and GST-claimable transactions in OFX 1.02 format for direct import into Xero.
  • MYOB (QIF) — exports the same transactions in QIF format for MYOB or Quicken. Category → account code mapping is configurable via business_export.account_codes in config.yaml.

Net Worth /net-worth

Complete balance sheet — bank account balances, manual assets (property, super, vehicles, investments), and liabilities (mortgage, HECS, car finance). The four summary cards show Bank Accounts, Manual Assets, Liabilities, and total Net Worth.

  • Bank Accounts — balance snapshots extracted automatically from ANZ Plus PDFs and Revolut CSVs during each import. The table shows the latest snapshot per account.
  • Net Worth Over Time — stacked area chart combining bank account history with manual asset snapshots and liabilities. Requires at least one data point to render.
  • Manual Assets — click + Add Asset to record property, superannuation, vehicles, or other investments. Enter a current value when adding; use + Snapshot on any asset row to record a new valuation date.
  • Superannuation — assets of type Superannuation accept additional fields: birth year, retirement age, expected annual return (%), and annual contribution. The Projected value at retirement is computed using compound interest (current balance + future contributions).
  • Property equity — link a mortgage liability to a property asset (using Linked Asset on the liability). The property row then shows Equity = value − mortgage and LVR %; teal <80%, amber 80–90%, red >90%.
  • Liabilities — click + Add Liability to record a mortgage, HECS/HELP, car finance, credit card, or other debt. Use + Snapshot to update the balance as you pay it down.
  • All manual asset and liability data is stored in Data/manual_assets.json.

Investment Portfolio /portfolio

Track share holdings by purchase lot — cost basis, unrealised P&L, and optional live market prices from Yahoo Finance.

  • Click + Add Lot to record a purchase: ticker symbol (e.g. VAS.AX for ASX, AAPL for NASDAQ), date, units, and cost per unit in AUD.
  • Holdings are aggregated by ticker — multiple lots of the same stock are summed to show total units, average cost, and total cost basis.
  • Click ↻ Refresh Prices to fetch the latest market price from Yahoo Finance (no API key required, internet access needed). Prices update the Market Value, Unrealised P&L, and Return % columns.
  • Portfolio data is stored in Data/portfolio.json. The Purchase Lots table at the bottom lists every lot individually.
  • For CGT calculations on disposals, see the Capital Gains page.
Configuration

Commitments /commitments

Track regular committed expenses such as rent, mortgage repayments, and annual subscriptions. Set the merchant name, amount, frequency, and due date. The app highlights upcoming commitments and flags them against imported transactions.

Bill Due Date Alerts Nav badge

An orange badge in the top navigation bar appears automatically whenever a commitment is due within the next 5 days. It shows the count of upcoming bills (e.g. ⏱ 2 bills due soon).

  • The badge is populated by GET /api/run-metrics on every page load — no separate action is required.
  • Click the badge or visit the Commitments page to see which specific bills are coming up, along with their due dates and amounts.
  • The alert uses the next_due date you have set on each commitment. Keep due dates current for accurate alerts.
  • Payments with a frequency of one-off are included if they fall within the 5-day window.

Subscriptions /subscriptions

A cost-focused view of your recurring commitments, sorted by monthly cost. Useful for auditing subscriptions and deciding what to cancel.

  • The summary grid at the top shows total Monthly Cost, Annual Cost, and counts of active vs cancelled items.
  • The active table shows each commitment with its monthly equivalent cost, annual equivalent, last charged date, and next due date.
  • Click Cancel to mark a commitment as cancelled — it moves to the Cancelled section at the bottom and is excluded from cost totals.
  • Click Restore to re-activate a cancelled commitment.
  • Changes sync with Data/commitments.json immediately — the Commitments page reflects the same data.

Budget Management Settings → Budgets

Set monthly spending limits per category. Budgets are stored in Data/budgets.json (not config.yaml) and managed entirely from this page.

  • Enter a monthly AUD limit for each category you want to track. Leave blank to exclude a category.
  • Click Load suggestions to auto-populate suggested limits based on your average monthly spend over the last 3 months, rounded up to the nearest $25.
  • Click Use on any row to copy the suggestion into the budget field. Click Use all to fill all rows at once.
  • Enable Rollover (checkbox column) for a category to carry any unspent balance forward into the next month. The rollover amount is capped at the base budget — you can never accumulate more than 2× the monthly limit. If you overspent last month, rollover adds nothing.
  • Click Save Budgets to persist all changes. The dashboard badge and budget status chips update immediately on the next page load.

Merchant Rules /settings/merchant-rules

Pattern-based rules that automatically assign a category to transactions matching a merchant name. Rules are applied on every import.

  • Add a rule by entering a merchant pattern (case-insensitive substring match) and selecting a category.
  • Delete a rule by clicking the × next to it.
  • Rules created here are saved to data/merchant_rules.json.

Accounts /settings/accounts

Manage display names and identifiers for each account. The system account name (as it appears in imported files) cannot be changed, but you can set a friendly name, institution, BSB, account number, or other identifier (card number, email, etc.) for each account.

  • Click Edit on any row to update an account’s details.
  • Click + Add Account to create a new account record manually before importing files for it.
  • Account records are seeded automatically the first time a file is imported for that account — you can then fill in the details here.

Reimbursements /reimbursements

Track business expenses submitted to your employer for reimbursement. Group unsubmitted expenses into a named batch, then link it to the reimbursement payment once received.

  • Select unsubmitted business expenses from the list and click Create Batch to group them into a claim.
  • Once payment is received, link the incoming transaction to the batch to mark it as settled.
  • Use Suggest Historical Matches to automatically pair past reimbursement payments with their expense groups.
  • Transactions must be flagged as Business Expense in the Transactions page before they appear here.

Superseded /superseded-pairs

Record confirmed provider switches so the AI recommendations engine never flags sequential replacements as duplicate spending. For example, if you switched health funds from HCF to Australian Unity, adding that pair here prevents the AI treating them as concurrent duplicate policies.

  • Enter the old provider name (as it appears in transaction descriptions) and the new provider that replaced it.
  • Optionally add a category and a short note for your own reference.
  • The AI also uses first_seen and last_seen dates from transaction history to detect sequential providers automatically — this page is for edge cases where the dates overlap during a transition period.
  • Changes take effect on the next import.

Coverage /coverage

Shows how many transactions were imported for each account in each calendar month, as a matrix. Gaps — months within an account’s active date range that have no transactions — are highlighted in red.

  • A red ! cell means no transactions were found for that account in that month. Check whether a statement file is missing for that period.
  • Dark cells indicate the month is outside the account’s active date range (before first transaction or after last transaction) — these are not gaps.
  • The amber alert at the top counts total gaps across all accounts; green means full coverage.

Capital Gains /capital-gains

Manual entry of investment sale summaries for each financial year. PFA does not auto-calculate cost base — use your broker or Sharesight annual statement and enter the summary figures here.

  • One card per financial year detected in your transaction history.
  • Short-Term Gains — proceeds from assets held less than 12 months. Fully taxable at your marginal rate.
  • Long-Term Gains — proceeds from assets held 12 months or more. Eligible for the 50% CGT discount (individuals and trusts).
  • The 50% Discount and Gains After Discount are calculated automatically: discount = long-term gains ÷ 2; after-discount = short-term + discounted long-term.
  • Capital Losses Applied — current-year or carried-forward losses offset against gains after discount.
  • Net Taxable Capital Gains is calculated automatically.
  • Carried Forward Losses — remaining losses to carry into the next FY.
  • Click Save. Saved data flows through to the FY Summary and the Tax Package export.

Franking Credits /franking-credits

Track dividend imputation credits received from Australian companies each financial year. Franking credits represent corporate tax already paid on your behalf and can reduce your personal income tax liability.

  • One card per financial year. Enter Cash Dividends Received and the Franking Credits associated with those dividends (both shown on your dividend statements).
  • The Grossed-Up Total (cash dividend + franking credit) is calculated automatically and represents your assessable income from dividends.
  • The optional Notes field accepts free text — useful for recording the source (e.g. "VAS ETF dividend").
  • Saved data appears on the FY Summary page in the Dividends & Franking Credits section and is included in the Tax Package ZIP export.
  • PFA uses the standard 30% company tax rate for imputation calculations. Partially-franked dividends: enter only the franking credit amount stated on your dividend statement (not the full imputed amount).
Planning

Cash Flow /cash-flow

Forward-looking projection of expected outflows over the next 30, 60, 90 days, or 12 months, with scenario modelling and one-off planned expenses.

  • Use the 30 / 60 / 90 days / 12 months toggle to switch the projection horizon. All cards and the chart update instantly.
  • Use the Optimistic / Baseline / Pessimistic scenario toggle to model variable spend at the 25th, 50th, or 75th percentile of your monthly history (computed from the last 12 months). Committed outflows are the same in all three scenarios.
  • Committed Outflows are sourced from your Commitments list — recurring items like rent, insurance, and subscriptions projected forward by their frequency and next due date.
  • Estimated Variable Spend is derived from your monthly non-committed spending history. The daily rate for the selected scenario is multiplied by the horizon days. Excludes income, transfers, investments, and amounts matching commitment merchants.
  • The Plotly chart shows three scenario lines (shaded region) for variable spend and a separate dashed line for manual one-off entries. The stepped red baseline is committed payments.
  • One-off Planned Expenses — click + Add one-off to record a known future expense (e.g. "Car service $800 in August"). These appear as a separate dashed line on the chart and are included in the Total Projected Spend card.
  • One-off entries are stored in Data/cashflow_entries.json. Click the ✕ button next to any entry to delete it.
  • Manage recurring commitments (add, edit, pause) on the Commitments page.

Financial Goals /financial-goals

Track progress towards financial milestones — emergency fund, holiday savings, debt repayment, investments, and more. Each goal has a target amount, current amount, regular contribution, and optional target date.

  • Click + Add Goal to create a new goal. Fill in the name, category, target amount, current amount, contribution amount and frequency, and an optional target date and linked account.
  • Progress bars show how close each goal is to its target. The estimated completion date is calculated automatically from your current amount and contribution frequency.
  • The Monthly Commitment summary card shows the total monthly equivalent of all active goal contributions, so you can see their impact alongside your committed outflows.
  • Click Edit on a goal to update the current amount as your balance grows — regular re-entry of the current balance is the primary way to track progress.
  • Click Delete to remove a goal permanently.
  • The search box filters goals by name or category. The goal list is sorted by nearest target date first.

Loans /financial-goals

Track money borrowed from or lent to others. Each loan record stores the direction, the other party, category filters that link it to the relevant transactions, and an optional note.

  • Click + Add Loan to create a record. Set Direction to I owe this (money you borrowed) or Owed to me (money you lent).
  • Repayment keyword — a word or phrase from the description of repayment transactions (e.g. mum). Transactions in the loan repayment categories matching this keyword are automatically linked to this loan.
  • Receipt keyword — a word or phrase from the description of the original receipt/disbursement transaction (e.g. lesley). Covers the other side of the loan automatically without manual action.
  • Once both keywords are configured, all related transactions on both sides are linked — no manual linking is needed.
  • The Loan Position card shows total received vs total repaid for each loan, calculated live from the database.
  • Click View Statement on any loan to see a full transaction-by-transaction ledger at /loans/<id>/statement.
  • The Unlinked Loan Transactions panel lists loan-category transactions not yet covered by any keyword — use it to spot transactions that need a keyword added.
AI

Proactive Insights Bell icon (nav)

A rule-based insight engine scans your recent transactions on every page load and surfaces up to 10 notable patterns via the 🔔 bell button in the top navigation bar. A red count badge appears when there are active insights.

  • Click the bell to open the insights tray, which slides in from the right. Click anywhere outside to close it.
  • Spend spikes — categories where this month’s spending is more than 50% above the 3-month average.
  • New merchants — merchants first seen in the last 14 days.
  • Large transactions — a single transaction ≥$100 in a non-excluded category within the last 30 days.
  • Weekend spending pattern — flagged if weekend spending averages more than 1.5× weekday spending over the last 90 days.
  • Excluded categories (never trigger insights): Income, Board & Lodging, Interest Income, Business Reimbursement, Family Loan Received, Transfers, Investment.
  • Insights are computed live from the database via GET /api/insights — no configuration required.

Merchant Analytics /merchants

A dedicated view of all merchants you’ve transacted with, aggregated from your full transaction history. Quickly identify where the most money goes, how often you visit, and whether visit frequency is changing.

  • The top three cards show the total number of unique merchants, total spend, and how many are currently visible after filtering.
  • Sort by Total Spend, Frequency (visit count), Avg Amount, or Last Seen using the buttons in the controls row.
  • Search by merchant name or category using the text box. The list filters live as you type.
  • Category filter — select a category from the dropdown to show only merchants in that category.
  • Each row shows: merchant name, category badge, visit count, average and max transaction size, last seen date, total spend bar, and a 6-month sparkline of spend over the last 6 calendar months.
  • Click a merchant name to jump to the Transactions page filtered to that merchant.
  • Transfer, investment, and income categories are excluded from the merchant list.

Debt Payoff Calculator /debt-payoff

An interactive calculator for modelling how quickly you can eliminate active borrowed loans using either the snowball or avalanche repayment strategy.

  • Snowball — pay minimums on all loans, then throw every extra dollar at the smallest outstanding balance first. Builds psychological momentum with quick wins.
  • Avalanche — pay minimums on all loans, then put all extra towards the highest interest rate first. Minimises total interest paid over time.
  • Set the Monthly Payment Budget using the number input or slider. The calculator auto-suggests a sensible starting value based on your current loan balances.
  • Each loan row shows its outstanding balance, monthly allocation, estimated interest cost, and projected payoff date. Dates update live as you adjust the budget or toggle the strategy.
  • Enter an Interest Rate % per loan and click Save to persist it to Data/loans.json. The rate persists across sessions. Personal loans with no interest charged should be left at 0%.
  • The Plotly chart shows the projected balance for each loan month by month until payoff under the selected strategy.
  • Only loans in the I owe this (borrowed) direction with an outstanding balance appear. Completed loans and lent-money records are excluded.
  • Loan records are managed on the Goals & Loans page.

Recommendations /recommendations

Full AI-generated spending analysis produced by Claude. Covers subscription audit, categories worth shopping around, spend spike analysis, dining vs groceries ratio, business expense summary, and five Quick Wins.

  • Select a period (All Time, This FY, Last FY, Last 12 Months, or Custom) and click ↻ Refresh to regenerate the analysis for that date range.
  • Regeneration calls the Anthropic API and typically takes 15–30 seconds.
  • The analysis respects your transaction categories — if a merchant is categorised as Utilities it will not be re-interpreted as something else based on its name.
  • The AI checks merchant date ranges before flagging same-category providers as duplicates — sequential provider switches are handled automatically.
  • Requires the ANTHROPIC_API_KEY Windows environment variable to be set (never stored in config.yaml).
Integrations

Data Sources /data-sources

Manage where your transaction data comes from. The page shows the freshness of each account and lets you connect bank feeds via the Basiq CDR (Consumer Data Right) integration.

  • Account staleness — each account card shows how many days since the last import. Green = fresh (≤7 days), amber = stale (8–30 days), red = very stale (>30 days). A badge in the top nav links here when any account is amber or red.
  • Connect via Basiq — click Connect a bank to open the Basiq consent flow in a new tab. After authorising, return here and the connection appears in the CDR Connections panel.
  • Sync — click Sync all to pull new transactions from all CDR connections. Transactions are deduplicated against existing imports using the same hash-based ID scheme as manual imports.
  • Disconnect — removes the CDR connection. Existing imported transactions are kept; only future syncing is disabled.
  • Mark as closed — use the toggle on an account card to hide a closed account from the staleness view without deleting its history.
  • Requires the BASIQ_API_KEY Windows environment variable to be set. If the key is missing, CDR features are hidden and a warning is shown. Set it with:
    [Environment]::SetEnvironmentVariable("BASIQ_API_KEY", "your-key", "User")
  • Account matching uses BSB + account number (dash-normalised). If no BSB match is found, the Basiq account name is used as a fallback.
