Distribution Foundation ✓
Infrastructure that must exist before sharing with any other user — all 9 items delivered
✅ Complete
Module toggle system M Done
src/modules.py — load/save Data/modules.json. Each module has a key, label, description, and default-on/off. Nav tabs and settings tabs filtered by enabled modules at render time. @require_module(key) decorator for route-level guard.
Enables: all future module toggles; first-run wizard
Settings > Modules page M Done
Toggle switches for all 10 modules: Budgets, Business & Tax, Investments, Loans, Goals, Pay-in-4 / PayPal, Transfers, Commitments, AI Recommendations, Coverage. Changes saved to Data/modules.json and applied immediately without restart.
Depends on: Module toggle system
First-run setup wizard M Done
Shown when no Data/modules.json exists. One page: "Tell us about your finances" — checkboxes for business income, PayPal, investments, loans, multiple accounts, AI recommendations. Sets modules config and API key field. Saves config, then redirects to dashboard.
Depends on: Module toggle system
Versioned schema migrations M Done
Replace ad-hoc init_db() with a schema_version table. Each migration is a numbered function _migrate_v2(conn), _migrate_v3(conn), etc. Applied in sequence on startup. Safe for users upgrading between app versions.
Structured logging S Done
Replace print() calls with Python logging module. Log file at Data/app.log with rotation (max 5 MB × 3 files). Levels: INFO for normal flow, WARNING for recoverable issues, ERROR for failures. Log path shown in Settings.
Startup config validation S Done
On server start, validate config.yaml: required keys present, account names match parsers, data directory writable, API key set if recommendations enabled. Show a clear in-app error banner for any failed check rather than silent failures mid-import.
Generic CSV importer with AI column detection L Done
User uploads any bank CSV; Claude Haiku inspects the first 10 rows and returns a JSON column mapping (date, description, debit, credit, balance, currency). Mapping saved as a named bank profile in Data/bank_profiles.json. Future uploads from same bank reuse profile — no AI call needed. UI for reviewing / adjusting the detected mapping before committing.
Pluggable AI backend M Done
Abstract the categorisation API call behind a backend interface. Support two backends: Claude API (current — requires API key) and Ollama (local LLM, zero API cost). Config: ai_backend: claude or ai_backend: ollama with ollama_model: llama3.2. Falls back to rule-only if neither configured.
Database encryption at rest M Done
The SQLite DB contains full financial history and should be encrypted if distributing. Options: SQLCipher (encrypted SQLite — requires recompile) or application-level encryption of the file at rest. Passphrase derived from the user's login password or a separate DB key stored in the OS keychain.

UX Modernisation ✓
Features expected by any user of a modern finance app — all 9 items delivered June 2026
✅ Complete
Mobile-responsive templates L Done
All pages work comfortably on a phone screen. Hamburger nav, stacked tables, touch-friendly tap targets, card-based layouts on narrow viewports. Base CSS breakpoints at 768px and 480px. Report pages (generated HTML) need a separate pass as they're not Jinja templates.
Dark / light theme toggle M Done
CSS custom properties for all colours in base.css. Light theme overrides via [data-theme="light"] on <html>. Toggle button in nav (sun/moon icon). Preference stored in localStorage and applied before first paint to avoid flash.
Transaction splitting M Done
Split one transaction into multiple category rows. E.g. $200 Coles → $150 Groceries + $50 Household. Stored as child rows in DB with parent_txn_id link and proportional amounts. Parent shown collapsed in transaction list; expand to see splits. Split rows categorised and reported individually.
Tags M Done
Cross-category labels for ad-hoc grouping (e.g. "Bali 2025", "Kitchen Reno", "Tax Deductible"). Stored as comma-separated field in DB. Tag filter in transactions page. Tag summary report showing total spend per tag. Tags are additive with categories — not a replacement.
Budget rollover M Done budgets
Option per category to carry unspent balance to next month. If Dining Out budget is $300 and March spend was $220, April's effective limit is $380. Rollover balance stored in Data/budgets.json alongside the base limit. UI shows base limit + rollover separately.
Subscription management view M Done
Dedicated page (distinct from Commitments) focused on subscriptions: service name, monthly cost, last charged, next expected charge, total spend YTD, cancellation notes field. Auto-detected from Subscriptions category + commitments detection. Manual add/edit/cancel-noted.
Auto-update checker S Done
On server start, check GitHub releases API for a newer version tag. If found, show a dismissible banner in the nav: "Version 2.1 available — see what's new". Cached for 24h so it doesn't hit GitHub on every restart. Only relevant for distributed builds.
Crash / error reporting (opt-in) S Done
Sentry integration, off by default, opt-in via config.yaml: error_reporting: true. Captures unhandled exceptions with stack trace and Flask route context. No PII or financial data sent — only code paths and error messages. For personal use this is the "copy this error" UI that appears in a modal when a route 500s.
Multi-currency display S Done
Revolut and Wise transactions stored in AUD but original currency/amount is in the note field. Parse and display it as a secondary line in transaction list: "AUD 47.83 (USD 31.50)". No recalculation — display only. Useful for reviewing FX transactions.

Financial Intelligence
Analysis and insight features that go beyond basic tracking
🔓 Unlock: competitive with or ahead of most consumer apps
Dashboard — savings rate & emergency fund M Done
Two headline KPIs added to the main dashboard: Savings rate (income minus expenses ÷ income, monthly trend sparkline) and Emergency fund runway (liquid savings balance ÷ avg monthly spend = months covered). Shown as large metric cards alongside existing balance summary.
Financial health score M Done
Composite 0–100 score shown on dashboard. Weighted components: savings rate (30%), emergency fund months (25%), budget adherence (20%), debt-to-income ratio (15%), net worth trend (10%). Drill-down panel shows component breakdown and specific improvement suggestions. Score history trended over 12 months.
Bill due date alerts M Done commitments
Extend Commitments from backward-looking detection to forward-looking alerts. Each detected recurring commitment gets a projected next-due date and estimated amount. Dashboard widget: "Upcoming bills in next 14 days — Electricity est. $185 in 3 days, Netflix $22.99 in 8 days". Alert badge in nav if any bill due within 5 days.
Proactive automated insights M Done recommendations
Triggered alerts that appear in a notification tray (bell icon in nav) without needing to run on-demand recommendations. Triggers: spend 30%+ above category average, new merchant first seen, budget hit 80%, largest single transaction this month, unusual weekend vs weekday pattern. Powered by rule engine first; Claude Sonnet for narrative if enabled.
Cash flow forecasting with known expenses M Done
Extend the 12-month cash flow projection to incorporate known future expenses: commitments projected forward, bills from due-date alerts, manual one-off entries ("holiday $3,000 in August"). Shows projected balance at end of each month accounting for these. Scenario toggle: optimistic / base / pessimistic (±10% income/spend variance).
Debt payoff calculator M Done loans
Snowball / avalanche calculator at GET /debt-payoff. Loans with outstanding balances displayed with per-loan interest rate input (saved via PATCH /api/loans/<id>/interest-rate). Total monthly budget slider allocates extra payment to priority loan. Plotly chart shows balance trajectories per method. payoff_months() and payoff_schedule() added to src/loans.py.
Merchant-level spend analytics S Done
Dedicated merchant analytics view at GET /merchants. Top merchants by total spend, visit frequency, average/max transaction size, last seen date, 6-month sparklines. Client-side search, category filter, and sort buttons. Merchant name links through to transactions page filtered by that merchant. SQL aggregates over DB; top 300 merchants passed as JSON constant to template.

Balance Sheet Completeness
Track all assets and liabilities for a true net worth picture
🔓 Unlock: complete financial picture beyond bank accounts
Net worth — manual asset entries M Done
Add/edit/delete assets (property, super, vehicle, investment, other) on the enhanced GET /net-worth page. Each asset has a snapshot history (+ Snapshot button). Data stored in Data/manual_assets.json via src/manual_assets.py. CRUD via POST/PATCH/DELETE /api/assets/<id> + POST /api/assets/<id>/snapshots.
Net worth — liabilities beyond loans S Done
Liabilities section on Net Worth page: mortgage, HECS/HELP, car finance, credit card, other. Each has type, interest rate, linked asset, and snapshot history. Stored in same Data/manual_assets.json file. CRUD via POST/PATCH/DELETE /api/liabilities/<id> + POST /api/liabilities/<id>/snapshots. Total liabilities subtracted in Net Worth summary card.
Net worth historical chart M Done
Stacked area Plotly chart on Net Worth page showing bank accounts + manual assets stacked, liabilities as a dotted negative fill, and net worth as a green line. Monthly LOCF (last observation carried forward) computed via compute_net_worth_history() in src/manual_assets.py.
Investment portfolio value tracking M Done investments
New GET /portfolio page with purchase lot recording (ticker, date, units, cost/unit). Holdings aggregated by ticker showing cost basis, avg cost, and unrealised P&L. Live price fetch via Yahoo Finance (no key, on-demand via ↻ Refresh Prices). Data in Data/portfolio.json. CRUD via POST /api/portfolio/lots + DELETE /api/portfolio/lots/<id>. Links to Capital Gains page.
Superannuation tracking S Done
Super implemented as an asset type on the Net Worth page with additional fields: birth year, retirement age, expected annual return %, and optional annual contribution. Projected balance at retirement computed via super_projected_balance() (compound interest + annuity FV formula). Shows projected value inline on the asset row.
Property equity calculator S Done
Property asset rows on Net Worth page show Equity and LVR % when a mortgage liability is linked via linked_liability_id. LVR colour-coded: teal <80%, amber 80–90%, red >90%.

Tax & Compliance (Australian)
Features that make the app genuinely useful at tax time
🔓 Unlock: replaces a spreadsheet for ATO reporting prep
CGT 50% discount eligibility flag S Done investments
Capital Gains page redesigned with separate Short-Term Gains (<12 months, fully taxable) and Long-Term Gains (≥12 months, 50% discount eligible) inputs. The 50% CGT discount and net taxable gains are auto-calculated. New short_term_gains / long_term_gains fields added to Data/capital_gains.json; backwards-compatible with legacy records.
Franking credits tracking M Done investments
New GET /franking-credits page for per-FY manual entry of cash dividends and franking (imputation) credits. Grossed-up total auto-computed. Data stored in Data/franking_credits.json. Included in FY Summary and tax export ZIP. New POST /api/franking-credits endpoint. Added to Investments nav tab group.
HECS / HELP repayment threshold alert S Done
HECS/HELP alert rendered inside each FY card on the FY Summary page. If taxable income exceeds the ATO minimum threshold, shows income, rate, and estimated compulsory repayment. Implemented via _fy_hecs_section() in src/reporter.py, using estimate_hecs_repayment() from src/tax_estimator.py.
Income tax estimation M Done business
Estimated Income Tax section added to each FY card on the FY Summary page. Shows gross tax, LITO reduction, Medicare levy, total payable, and net income after tax. A secondary row shows the adjusted estimate if all recorded GST-claimable and tax-deductible expenses are applied. Implemented via _fy_income_tax_section() in src/reporter.py and src/tax_estimator.py (Stage 3 + pre-Stage-3 bracket functions).
Tax summary export M Done
Download Tax Package button on each FY Summary card. GET /api/tax-export/<fy> returns a ZIP containing summary.txt (income, deductions, estimated tax, HECS), gst_claimable.csv, business_expenses.csv, tax_deductions.csv, capital_gains.csv, franking_credits.csv, and README.txt. Accountant-ready handoff without app access.

Ecosystem & Integration
Connections to external systems and power-user features
🔓 Unlock: professional-grade and household-shareable
PyInstaller / Briefcase bundle M Done
pfa.spec + build_windows.ps1; --onedir output in dist/PersonalFinanceAnalyser/ (154 MB, 23.6 MB exe). Auto-opens browser on launch. _BUNDLE_DIR / BASE_DIR split in server.py — read-only assets (templates, static, docs, src) from bundle; user data (config.yaml, Data/) from exe directory. Run .\build_windows.ps1 to rebuild after code changes.
CDR / open banking integration XL Planned
Australian Consumer Data Right via Basiq or Frollo API — live transaction feeds from any CDR-accredited bank without CSV downloads. Requires ADR accreditation (complex, costly for a distributed product). For personal use: Basiq has a free developer tier. Transactions fetched daily, deduped by provider transaction ID, fed into existing pipeline.
Xero / MYOB export M Done business
src/biz_export.py; GET /api/business-export/<fy>?format=ofx|qif|csv; filters to is_business + is_gst_claimable transactions; OFX 1.02 for Xero, QIF for MYOB; category → account code mapping via business_export.account_codes in config.yaml. "Xero (OFX)" and "MYOB (QIF)" download buttons on each FY Summary card.
REST API / webhook system L Planned
Documented REST API for power users: GET /api/v1/transactions, GET /api/v1/summary/monthly, POST /api/v1/transactions/categorise. API key auth. Webhook support: fire a POST to a configured URL on new transaction import, budget exceeded, or goal milestone hit. Enables Notion/spreadsheet/Zapier integration.
Multi-user / household sharing XL Planned
Allow a second user (partner) to access the same data with their own login. Roles: Owner (full access) and Viewer (read-only) or Co-owner (full access). Each user sees the same transactions but can have their own category overrides. Shared budgets with household totals. Requires proper session management and user table in DB.

Family Loan transfer ↔ loan auto-link S Done loans
When confirming a "Family Loan" transfer pair, auto-add the txn_ids to the matching loan record's linked_receipt_txn_ids or linked_repayment_txn_ids. Currently requires manual linking on the Goals & Loans page after confirmation.
Upload modal account selector S Done
When a file is uploaded and the account cannot be determined by content-sniffing or filename pattern, show a per-file account dropdown in the upload modal rather than failing silently. Less urgent since content-sniffing covers most cases.
