T42 Split a transaction via the API and verify SPLIT badge appears
▶

Note: split initiation is currently API-only — no UI button exists yet. Use browser devtools console or curl.

  1. 1Find a transaction ID to split (hover its description — the txn_id is shown in the Change modal title, or check the URL when the modal is open). Use a transaction with a known amount, e.g. -$100.00.
  2. 2In browser devtools console, run:
    fetch('/api/txn/YOUR_TXN_ID/split', {method:'POST', headers:{'Content-Type':'application/json'}, body: JSON.stringify({splits:[{description:'Part A',amount:-60,category:'Groceries'},{description:'Part B',amount:-40,category:'Dining Out'}]})}).then(r=>r.json()).then(console.log)
  3. 3Confirm the response contains "ok": true and two child IDs.
  4. 4Re-generate reports (upload a file or wait for the next pipeline run), then reload the Transactions page.
  5. 5Find the parent transaction — confirm a green SPLIT badge appears next to the description.
Expected: API returns ok: true. After report regeneration, parent row shows SPLIT badge. The two child transactions do NOT appear as separate rows in the default view (they're hidden inside the parent).
