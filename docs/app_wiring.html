Key Data Flows
Flow Path
Upload & Import (auto-pipeline) Browser → POST /api/upload → Raw Data/ → _run_pipeline_bg() spawns subprocess → parsers → enricher → categoriser (rules → cache → Haiku) → finance.db → reporter (parallel) → HTML reports. Pipeline log → Data/pipeline.log
Recommendations finance.db → server → Anthropic Sonnet 4.6 API → HTML response → browser
Category / Sub-category Edit Browser → PATCH /api/transactions/<id> → finance.db (saves instantly). Sub-categories served from GET /api/subcats (merges src/utils.py SUBCATS + Data/custom_subcats.json). New sub-categories added via POST /api/subcats/add → Data/custom_subcats.json.
Pay-in-4 Detection finance.db → payin4_detector.py (purchase-first: find 4 instalments at 14-day intervals, verify sum, link to ANZ debits) → Data/payin4_groups.json → reporter.py embeds in transactions.html as expandable panel per purchase row
Account Management Browser → GET/PUT/POST /api/accounts → finance.db accounts table. Seeded automatically from distinct transaction account names on server start.
Live Page Render (Dashboard, FY Summary, Net Worth, Review, Transfers) Browser → GET /dashboard (etc.) → Flask route calls prepare_*_data() from reporter.py → loads from finance.db + JSON files → render_template() → Jinja2 template (templates/*.html) → browser. Always shows current data. Static counterparts in reports/ still generated by reporter.py for CLI / Refresh Charts workflow.
Tax Export (ZIP) Browser → GET /api/tax-export/<fy> → server assembles ZIP from finance.db (income, expenses, business rows, CGT data) + Data/franking_credits.json → returns .zip download to browser
Xero / MYOB Export Browser → GET /api/business-export/<fy>?format=ofx|qif|csv → server → biz_export.py (reads is_business / is_gst_claimable rows from finance.db; category → account code from config.yaml) → returns OFX (Xero), QIF (MYOB/Quicken), or CSV file to browser
